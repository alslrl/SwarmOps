# SwarmOps — 랄프톤 서울 중간발표 Outline

## Slide 1: 표지
- **프로젝트명**: SwarmOps
- **부제**: 800개 AI 구매자가 당신의 상품 전략을 검증합니다
- **팀명**: SwarmOps
- **팀원**: 손세호
- **GitHub**: github.com/alslrl/SwarmOps

## Slide 2: 문제 정의 — "적용하기 전에 미리 확인할 수 있다면?"
- **누구의 문제?**: 이커머스 셀러 (쿠팡, 올리브영 등 온라인 판매자)
- **현실**:
  - 셀러는 상품 제목, 카피, 가격을 수시로 바꿔야 한다
  - 하지만 **어떤 조합이 경쟁사를 이기는지** 적용하기 전까지 알 수 없다
  - A/B 테스트는 느리고, 감으로 바꾸면 매출이 떨어질 수도 있다
- **질문**: **"실제 시장에 내보내기 전에, 이 전략이 먹히는지 먼저 확인할 수 있다면?"**
- **배경**: 탈모샴푸처럼 경쟁사 3~4개가 같은 카테고리에서 싸우는 레드오션 — 작은 차이가 순위를 결정하는 시장

## Slide 3: 솔루션 — SwarmOps
- **한 문장 정의**: "출시 전에, 800명의 AI 구매자로 내 상품 전략이 경쟁사를 이기는지 미리 검증하는 대시보드"
- **영감**: Mirofish — 대규모 AI 에이전트에 실제 소비자 페르소나를 부여하면, 인간의 구매 결정을 시뮬레이션할 수 있다
- **핵심 접근**:
  - 한국 소비자 8개 아키타입 (가격민감형, 프리미엄, 신뢰우선형 등)
  - 각 아키타입에서 800명의 개별 에이전트 생성 (고유 이름, 성격, 예산대)
  - 각 에이전트가 독립적으로 LLM을 통해 "이 상품 살까 말까" 판단
  - 반복 시뮬레이션 + 홀드아웃 검증으로 신뢰성 확보

## Slide 4: 핵심 워크플로우 / 데모
- **사용자 흐름**:
  1. 대시보드 접속 → 현재 상품 정보 확인 (트리클리닉 탈모샴푸 vs 경쟁사 3개)
  2. 제목 / 카피 / 가격 수정 (예: 29,900원 → 19,900원)
  3. "RUN SIMULATION" 클릭
  4. 800명의 AI 구매자가 실시간으로 선택하는 모습을 파티클 애니메이션으로 관찰
  5. 결과: 최적 전략 추천 + before/after Diff + 홀드아웃 수익 검증
- **시각적 요소**: 3-패널 대시보드 (입력 / 시뮬레이션 파티클 플로우 / 결과), 다크 테마

## Slide 5: 나의 랄프 세팅 / 랄프 역량
- **운영 방식**: Ouroboros 루프를 활용한 자동 반복 빌드
  - PRD 작성 → Seed 생성 → Test Spec 자동 도출 → 빌드 → 검증 → 진화 반복
  - Claude Code + Ouroboros로 1인이 5시간 만에 0→1 프로덕트 완성
- **AI 활용**:
  - gpt-5.4: 전략 후보 생성 + 현실성 판단 (상인 관점)
  - gpt-5-nano: 800개 에이전트 개별 구매 평가 (병렬 200+ 동시 호출)
  - Claude Code: 프로덕트 빌드 전체를 Ouroboros 루프로 자동화
- **핵심 역량**: PRD → Seed → Test Spec → 자동 빌드/검증의 반복 루프로 단독 개발 가속
